# 电动滑板 App 需求文档（iOS 优先）

## 0. 已确认的商业阶段范围

正式用户品牌统一为 **BM**。

第一阶段商业 MVP 仅包含：

- BLE 设备搜索、连接、断线与重连
- 实时速度、电量、里程、功率、温度等设备数据
- 设备身份、型号、固件版本和连接状态管理
- 用户可理解的故障与安全提示
- 语言、单位等基础设置
- 隐私政策、用户协议、支持和开源合规入口

第二阶段再实现：

- 固件升级
- 用户账号、登录与 Token
- 云端设备绑定
- 骑行排行榜
- 社交或社区能力

第一阶段产品页面必须通过受控的产品模型访问设备能力，不允许直接调用电流、占空比、转速、终端、配置写入、Bootloader 或原始固件上传等工程接口。底层 BLE、Packet 和 Commands 协议保持兼容。

## 1. 项目背景

本项目基于当前 BMESC APP 代码库进行产品化改造。当前代码库来自 vesc_tool 分支，已经具备电机控制器通信、蓝牙连接、实时数据读取、配置管理和固件升级等底层能力。

新的产品目标不是继续作为工程调试工具，而是面向电动滑板/滑板车用户，形成自有品牌 App、云端服务和后台管理系统。

## 2. 开发优先级

本项目优先开发 iOS 应用。

开发顺序建议如下：

1. iOS App MVP
2. 服务端 API
3. 后台管理 Web
4. Android App 同步适配

Android 版本仍属于完整产品范围，但不作为第一阶段优先目标。Android 后续应尽量复用 iOS 阶段形成的 QML 页面、C++ 通讯能力、服务端接口和产品流程。

## 3. 当前代码能力评估

### 3.1 已具备能力

当前代码已经具备以下可复用能力：

- BLE 蓝牙扫描与连接

### 3.2 当前不足

- 骑行排行榜
- 服务端固件版本管理
- 服务端 App 版本管理

- App Store 上架所需的品牌、隐私、权限说明整理

## 4. 产品目标

### 4.1 iOS App 目标

iOS App 应成为用户连接、查看和管理电动滑板设备的主要入口。

核心目标：

- 用户可以通过蓝牙连接自己的设备
- 用户可以查看设备实时状态
- 用户可以查看累计里程、最高速度等骑行信息
- 用户可以查看和管理本地设备信息
- 用户可以收到明确的故障和安全提示
- 用户可以在中英文之间切换
- App 满足 App Store 上架要求

账号、云端绑定、固件升级和排行榜属于第二阶段。

### 4.2 服务端目标

服务端为 iOS App、后续 Android App 和后台管理系统提供统一接口。

核心目标：

- 用户账号管理
- 设备绑定关系管理
- 骑行数据统计
- 排行榜数据计算
- App 版本更新管理
- 固件版本与固件包管理
- 后台管理接口

### 4.3 后台管理目标

后台管理系统用于运营人员查看用户、设备、骑行统计，并管理 App 包和固件包。

核心目标：

- 管理员登录
- 用户管理
- 设备管理
- 骑行数据查看
- 排行榜统计
- App 软件包上传
- 固件包上传与发布

## 5. 功能需求

## 5.1 iOS App 功能需求

### 5.1.1 用户模块

用户注册：

- 用户使用手机号注册
- 通过短信验证码完成验证
- 首次注册后设置用户昵称

用户登录：

- 用户使用手机号和短信验证码登录
- 登录后保存 Token
- 支持退出登录
- 后续需支持账号注销，以满足 App Store 审核要求

我的账号：

- 显示手机号
- 显示昵称
- 支持修改昵称
- 显示隐私政策和用户协议

### 5.1.2 设备连接模块

设备搜索：

- App 启动后可搜索附近蓝牙设备
- 显示设备名称和连接状态
- 优先展示已绑定设备

设备连接：

- 用户点击设备后建立 BLE 连接
- 连接成功后读取设备基础信息
- 首次连接时引导用户绑定设备

设备绑定：

- 绑定设备到当前登录用户
- 绑定信息同步到服务器
- iOS 不应依赖 BLE MAC 地址作为唯一标识
- 建议优先使用控制器 UUID、序列号或固件提供的稳定设备 ID

设备解绑：

- 用户可在“我的设备”中解绑设备
- 解绑后服务端同步删除用户与设备绑定关系

### 5.1.3 首页仪表盘

首页展示普通用户最关心的信息：

- 当前速度
- 电量
- 累计里程
- 本次里程
- 最高速度
- 设备连接状态
- 故障/异常提醒

当前代码已能读取电压、电流、功率、温度、故障、速度相关统计等数据，但需要重新包装为用户友好的界面。

### 5.1.4 骑行统计

本地显示：

- 本次骑行时长
- 本次骑行里程
- 平均速度
- 最高速度
- 累计里程

云端同步：

- App 定时或在骑行结束/断开连接时上传累计里程和骑行摘要
- 服务端按用户、设备和时间维度保存统计数据

### 5.1.5 排行榜

排行榜展示：

- 总里程排行榜
- 月度排行榜
- 周排行榜

排行榜数据来源：

- 服务端根据用户上传的骑行数据计算
- App 只展示服务端返回结果

### 5.1.6 设备信息

设备详情页显示：

- 设备名称
- 序列号或 UUID
- 设备型号
- 固件版本号
- 软件/硬件信息
- 最近连接时间
- 绑定状态

### 5.1.7 固件升级

固件升级流程：

1. App 连接设备
2. 读取当前设备型号和固件版本
3. 请求服务端查询最新固件
4. 下载固件包
5. 校验固件 hash
6. 调用现有固件升级流程写入设备
7. 升级完成后重新读取固件版本

iOS 重点验证：

- 升级过程中蓝牙断连
- 升级过程中 App 进入后台
- 升级过程中锁屏
- 固件包下载失败
- 固件型号不匹配

### 5.1.8 App 更新

iOS App 更新不应绕过 App Store。

服务端可提供版本检测接口：

- 当前最新版本
- 是否强制更新
- 更新说明
- App Store 下载链接

App 检测到新版本后跳转 App Store。

### 5.1.9 多语言

第一阶段支持：

- 中文
- 英文

多语言范围：

- 首页
- 连接流程
- 我的设备
- 登录注册
- 设置
- 固件升级
- 错误提示
- 隐私政策和用户协议入口

### 5.1.10 设置

设置页包含：

- 语言切换
- 单位设置
- 固件升级入口
- App 版本
- 隐私政策
- 用户协议
- 关于我们
- 退出登录
- 注销账号

工程调试功能应默认隐藏，可通过开发者模式或内部版本开启。

## 6. 分层实现说明

### 6.1 平台/build 层

iOS 优先涉及：

- iOS Bundle ID
- iOS Display Name
- iOS App Icon
- iOS Launch Screen
- iOS Info.plist 权限说明
- Apple Developer 账号
- 证书和 Provisioning Profile
- TestFlight 测试
- App Store 上架材料

当前项目中 iOS 构建已有基础，但 target 仍有 VESC Tool 残留，需要品牌化替换。

### 6.2 C++ 后端/通信层

应尽量复用并保持稳定：

- BLE 通信
- 协议解析
- 实时数据读取
- 固件写入
- 设备配置读取

除非明确涉及协议变化，不应随意修改通信协议语义。

### 6.3 QML/UI 层

需要重点重做：

- 登录注册页
- 产品化首页
- 设备连接页
- 我的设备页
- 设备详情页
- 排行榜页
- 设置页
- 固件升级页

现有工程页面和高级配置入口应隐藏，不作为普通用户默认流程。

### 6.4 资产/品牌层

需要替换：

- App 名称
- iOS 图标
- 启动图
- About 页面
- 隐私政策链接
- 用户协议链接
- 支持邮箱
- 官网链接
- 用户可见 VESC 字样

### 6.5 设备预设/配置层

后续应加入自有硬件预设：

- 默认设备型号
- 默认电池参数
- 默认速度/功率限制
- 默认固件适配关系
- 产品化配置向导

第一阶段不建议大范围改协议或底层配置逻辑。

## 11. 主要风险

- iOS BLE 不暴露硬件 MAC 地址，设备绑定必须使用稳定设备 ID
- App Store 审核会严格检查蓝牙用途、数据采集和账号注销
- 固件升级涉及硬件安全，必须加入型号校验和 hash 校验
- 当前代码仍有工程工具和 VESC 品牌残留，上架前必须清理
- 后台和云端不是现有代码能力，需要单独开发
- Android 后续适配应避免反向影响 iOS 已稳定流程
