# BM 开源移动 App PRD（iOS + Android）

## 1. 文档目的

本 PRD 用于指导基于当前 BMESC APP / VESC Tool 开源代码的 BM 移动 App 产品化改造。目标是在保留成熟 BLE、Packet、Commands 通讯能力的前提下，重做面向普通用户的移动端 UI、交互流程和产品模型，形成可上架 App Store 与安卓应用市场的开源 App。

本文件聚焦当前商业 MVP，不包含账号、云端绑定、排行榜、社交、固件升级等第二阶段能力。

## 2. 产品定位

### 2.1 产品名称

- 用户可见品牌：BM
- App Store / 安卓市场展示名称：BM
- 不在 App 名称、图标、启动页、主界面、设置页、公开营销文案中使用 VESC 商标。
- 仅在开源许可、兼容性说明或第三方致谢中保留必要来源说明。

### 2.2 目标用户

- BM 硬件产品的普通用户。
- 不熟悉电机控制器、电流、占空比、RPM、终端命令、固件烧录等工程概念的用户。
- 需要快速连接设备、查看状态、理解故障提示、获得支持的用户。

### 2.3 产品原则

- 操作简单，默认自动发现设备。
- 首页优先显示用户关心的数据：速度、电量、里程、最高速度、故障状态。
- 工程能力保留在代码中，但不出现在商业 MVP 导航中。
- 产品 QML 不直接调用 `Commands`、可变 `ConfigParams`、终端、Bootloader 或原始固件上传接口。
- BLE、Packet、Commands、VescInterface 协议语义保持稳定，除非单独立项修改。

## 3. MVP 范围

### 3.1 必须实现

- BLE 设备发现与连接。
- 自动开始 BLE 扫描。
- 下拉重新扫描。
- 显示附近设备列表。
- 发起 BLE 连接。
- 显示连接中状态。
- 15 秒连接超时提示。
- 显示 BLE 错误。
- 连接成功后更新全局连接状态。
- 根据连接设备类型进入对应设备信息与实时数据流程。
- 显示固件版本号、软件版本号、UUID。
- 显示实时速度、电池电量百分比、里程、最高时速、故障提示。
- 实时数据页高频轮询，其他页面低频轮询。
- 超过 2 秒未收到遥测数据时标记数据失效，避免继续展示过期数值。
- 退出 App 前断开 BLE 设备。
- 设置、隐私、法律、支持、开源合规入口。
- iOS 与 Android 均作为目标平台，首轮验证优先 iOS。

### 3.2 明确不属于首版

- 固件更新。
- 用户账号、登录、注册、Token。
- 云端设备绑定。
- 社区、动态、排行榜、挑战活动。
- 社交分享、评论、点赞。
- 终端命令。
- 电机电流、占空比、RPM 控制。
- 原始配置写入。
- Bootloader 操作。
- 不受控的固件上传。

## 4. 设计参考

### 4.1 原型来源

当前可参考原型文件：

- `vesc_premium_mobility_prototype.html`

该原型可作为视觉风格和页面层级参考，但需要按 BM 商标规则清理 VESC、Premium Mobility、VIP 调参、云端同步等不属于当前 MVP 的文案。

### 4.2 视觉方向

- 深色背景。
- 金色主按钮与状态强调。
- 大号速度仪表。
- 卡片式信息区。
- 页面数量少，导航清楚。
- 中文优先，保留英文切换能力。

### 4.3 交互方向

- App 启动后自动进入设备发现逻辑。
- 普通用户不需要理解 BLE 技术细节。
- 出错时给出可执行建议，例如“请确认车辆已开机并靠近手机”。
- 连接成功后自动进入设备信息或实时数据，不让用户在工程页面中选择。

## 5. 信息架构

首版固定保留 3 个底部 Tab：

- 首页
- 设备
- 我的

不建议在首版使用“发现”作为底部 Tab 名称。“发现”容易被理解为社区、内容流、活动或排行榜入口，与商业 MVP 范围不一致。

### 5.1 首页

用途：

- 展示连接状态。
- 展示主要实时数据。
- 作为进入设备连接流程和实时详情的主入口。

内容：

- 当前连接状态：未连接、扫描中、连接中、已连接、连接失败。
- 设备名称。
- 实时速度。
- 电池电量百分比。
- 累计里程。
- 最高时速。
- 故障状态摘要。
- 主操作按钮：连接设备 / 查看实时数据 / 断开。

职责边界：

- 首页可以展示扫描、连接、断连、数据失效等状态摘要。
- 首页不直接承载完整 BLE 扫描列表，避免与设备 Tab 重复。
- 用户点击“连接设备”时跳转到设备 Tab 的扫描列表。
- 已连接时，首页点击“查看实时数据”进入实时数据详情页。
- 已连接时，首页点击“断开”触发产品连接模型断开，完成后保持在首页未连接态。

### 5.2 设备

用途：

- 承载设备发现、连接、节点选择和设备详情。
- 作为所有设备管理相关页面的唯一入口。

内容：

- BLE 扫描列表。
- 下拉刷新。
- 已发现设备。
- 连接中状态。
- BLE 错误。
- Express 模块下的 CAN 节点列表。
- 当前连接设备详情。

职责边界：

- 设备 Tab 可以发起扫描、连接、重试、断开、切换 CAN 节点。
- 设备 Tab 不展示社区、排行榜、挑战、动态、登录或云绑定入口。
- 连接成功后，设备 Tab 根据设备类型进入 UART 直连详情或 Express 节点选择。
- 当用户从设备 Tab 返回首页时，连接状态和遥测状态必须保持一致。

### 5.3 我的

用途：

- 语言切换。
- 支持入口。
- 隐私政策。
- 用户协议。
- 开源许可。
- 关于BM。

不显示登录、退出登录、云同步、账号资料等第二阶段入口。

### 5.4 页面职责总表

| 页面 | 首版职责 | 不应承担 |
| --- | --- | --- |
| 首页 | 当前连接摘要、核心实时数据、查看实时详情、连接入口、断开入口 | 完整扫描列表、CAN 节点管理、工程调试 |
| 设备 | BLE 扫描、连接、失败重试、Express 节点选择、设备详情、节点切换 | 社区内容、排行榜、账号绑定、固件升级 |
| 实时数据详情 | 大号速度仪表、实时遥测、故障提示、数据失效提示 | 原始参数写入、电流/RPM/占空比控制、终端 |
| 我的 | 设置、语言/单位、支持、隐私、法律、开源许可、关于 BM | 登录注册、云同步、VIP、社交资料 |

## 6. 核心用户流程

### 6.1 首次启动与自动扫描

1. 用户打开 App。
2. App 检查蓝牙权限和蓝牙状态。
3. 若权限允许，自动开始 BLE 扫描。
4. 默认停留在首页，首页显示“正在搜索附近 BM 设备”的摘要状态。
5. 扫描结果实时进入设备 Tab 的设备列表。
6. 用户点击首页“连接设备”或底部“设备”Tab 后进入扫描列表。
7. 用户可在设备 Tab 下拉重新扫描。
8. 用户点击设备后发起连接。

异常：

- 蓝牙未开启：提示用户打开蓝牙。
- 没有权限：提示用户前往系统设置授权。
- 未发现设备：提示用户确认车辆已开机并靠近手机。
- 权限或蓝牙状态恢复后，允许用户在设备 Tab 手动重试扫描。

### 6.2 BLE 连接

1. 用户点击设备。
2. UI 进入连接中状态。
3. 停止或弱化扫描，避免影响连接稳定性。
4. 连接计时开始。
5. 15 秒内连接成功则进入连接成功流程。
6. 超过 15 秒未成功则显示连接超时。
7. BLE 返回错误时显示用户可理解的错误消息。

验收标准：

- 连接中不可重复点击造成多个并发连接。
- 15 秒超时后状态回到可重试。
- 失败后用户可以立即重新扫描或重新连接。
- 错误文案不暴露底层栈信息，详细错误可记录到日志。
- 连接失败、连接超时或用户取消连接后，页面停留在设备 Tab 的扫描列表，并保留可重试操作。

### 6.3 连接 Express 模块

判定条件：

- BLE 连接成功后，蓝牙设备名称识别为 Express 模块。

流程：

1. 更新全局连接状态为已连接。
2. 读取 Express 模块基础信息。
3. 扫描或读取 CAN 节点列表。
4. UI 展示 CAN 节点上的设备列表。
5. 用户点击某个节点。
6. 读取并显示该节点的固件版本号、软件版本号、UUID。
7. 进入该节点对应的实时数据页面。
8. 按 CAN 节点 ID 记录上次选择的节点。
9. 下次打开 App 并连接同一 Express 模块时，若节点列表中仍存在该节点 ID，则默认自动选择上次节点，不要求用户手动选择。
10. 若用户从实时数据页返回设备 Tab，可以切换节点或重新扫描节点。

验收标准：

- 未选择节点前不展示错误节点的遥测数据。
- CAN 节点列表为空时给出明确提示。
- 上次保存的节点 ID 不存在或不可用时，回到节点选择列表。
- 自动选择节点需要允许用户切换设备。
- 切换节点前需停止或隔离当前节点遥测，避免显示上一个节点的旧数据。
- Express 模块断开后，CAN 节点列表和当前节点遥测均标记为无效。

### 6.4 连接 UART 直连设备

判定条件：

- BLE 连接成功后，内部兼容识别蓝牙名称固定为 `VESC BLE UART`。
- 用户界面不直接展示上述兼容名称，优先显示 BM 设备名、设备别名或通用“已连接设备”。

流程：

1. 更新全局连接状态为已连接。
2. 读取并显示固件版本号、软件版本号、UUID。
3. 不进入 CAN 节点选择。
4. 直接进入实时数据页面。

验收标准：

- 用户不需要理解 CAN 节点。
- 连接成功后 2 秒内开始请求设备信息。
- 设备信息未读到时显示“读取中”或“暂不可用”，不显示假数据。
- 连接成功后底部 Tab 仍可切换，返回首页时展示同一台设备的摘要状态。

### 6.5 实时数据页面

展示字段：

- 实时速度。
- 电池电量百分比。
- 累计里程或当前里程。
- 最高时速。
- 故障提示。
- 连接状态。
- 数据更新时间或数据有效状态。

轮询策略：

- 实时数据页：高频轮询。
- 其他页面：低频轮询。
- 建议首版默认值：
  - 实时页：50 ms 至 250 ms，根据设备稳定性确定最终值。
  - 非实时页：250 ms 至 1000 ms。
  - 超过 2 秒未收到遥测数据：`telemetryValid = false`。
- 最高时速为本次连接会话内的最大速度，由产品模型根据实时速度计算；断开或重新建立连接后重置。

数据失效表现：

- 速度显示为 `--` 或变灰。
- 电量、里程、最高速度显示为 `--`。
- 显示“数据暂未更新，请检查连接”。
- 不继续展示旧数值作为当前状态。

验收标准：

- 离开实时页后轮询频率降低。
- 断开连接后实时数据清零或标记无效。
- 故障状态使用普通用户可理解文案。
- 从实时数据页返回首页或设备 Tab 不应触发断连。

### 6.6 退出 App 前断开

要求：

- App 主动退出或进入销毁流程时，断开 BLE 连接。
- iOS 与 Android 均需要处理应用生命周期。
- App 进入后台时立即主动断开 BLE，避免后台占用设备连接。

验收标准：

- 退出后设备不保持异常连接占用。
- 再次打开 App 可以重新扫描和连接。
- 从后台恢复到前台后默认回到未连接/扫描流程，不展示假连接状态。

### 6.7 导航状态机

首版必须使用单一产品连接状态驱动页面跳转，避免多个 QML 页面各自维护连接状态。

| 状态 | 默认页面 | 用户可见内容 | 允许操作 | 下一状态 |
| --- | --- | --- | --- | --- |
| 权限检查中 | 首页 | 正在检查蓝牙权限 | 等待系统结果 | 扫描中 / 权限被拒绝 / 蓝牙未开启 |
| 权限被拒绝 | 设备 | 授权说明和前往系统设置提示 | 打开系统设置、重试 | 权限检查中 / 扫描中 |
| 蓝牙未开启 | 设备 | 打开蓝牙提示 | 重试 | 权限检查中 / 扫描中 |
| 扫描中 | 首页摘要 + 设备列表 | 正在搜索附近 BM 设备 | 进入设备 Tab、下拉刷新、点击设备 | 发现设备 / 未发现设备 / 连接中 |
| 未发现设备 | 设备 | 未找到设备和排查建议 | 重新扫描 | 扫描中 |
| 发现设备 | 设备 | BLE 设备列表 | 点击设备连接、下拉刷新 | 连接中 |
| 连接中 | 设备 | 当前设备连接中 | 取消连接、等待结果 | 已连接待识别 / 连接失败 / 连接超时 |
| 连接失败 | 设备 | 用户可理解错误文案 | 重试连接、重新扫描 | 扫描中 / 连接中 |
| 连接超时 | 设备 | 15 秒超时提示 | 重试连接、重新扫描 | 扫描中 / 连接中 |
| 已连接待识别 | 设备 | 正在读取设备类型和基础信息 | 等待读取、断开 | UART 直连 / Express 节点选择 / 识别失败 |
| UART 直连 | 实时数据详情 | 设备信息和实时数据 | 返回首页、进入设备详情、断开 | 已连接 / 断开中 |
| Express 节点选择 | 设备 | CAN 节点列表 | 选择节点、重新扫描节点、断开 | 节点读取中 / 断开中 |
| 节点读取中 | 设备 | 正在读取节点信息 | 等待读取、返回节点列表、断开 | 节点实时数据 / 节点读取失败 |
| 节点实时数据 | 实时数据详情 | 所选节点实时数据 | 返回首页、切换节点、断开 | Express 节点选择 / 断开中 |
| 已连接 | 首页 | 当前设备摘要和实时数据摘要 | 查看实时数据、设备详情、断开 | 实时数据详情 / 设备 / 断开中 |
| 数据失效 | 首页或实时数据详情 | 数据暂未更新提示 | 等待恢复、断开、重新连接 | 已连接 / 断开中 / 扫描中 |
| 断开中 | 当前页 | 正在断开 | 等待结果 | 未连接 |
| 未连接 | 首页 | 未连接摘要 | 连接设备 | 扫描中 |

导航规则：

- 自动扫描可以在首页启动，但完整列表只在设备 Tab 展示。
- 首页、设备 Tab 和实时数据详情必须读取同一个产品连接状态。
- 任何连接失败、连接超时、识别失败或节点读取失败都回到设备 Tab，并保留重试入口。
- 断开成功后回到首页未连接态，并清空或置灰遥测数据。
- 用户主动切换 Tab 不应改变 BLE 连接；只有点击断开、退出 App 或平台生命周期策略触发断连。
- 若 App 冷启动时存在上次 Express 节点 ID 记忆，仍需先连接同一 Express 模块并验证该节点 ID 可用，再自动进入节点实时数据。

## 7. 页面需求

### 7.1 扫描页

归属：

- 扫描页属于设备 Tab，不作为独立底部 Tab。
- 首页仅展示扫描摘要和进入扫描页的按钮。

状态：

- 权限检查中。
- 蓝牙未开启。
- 扫描中。
- 未发现设备。
- 发现设备。
- 连接中。
- 连接成功。
- 连接失败。
- 连接超时。

设备列表字段：

- 设备名称。
- 信号强度，如当前 BLE 层可提供。
- 最近发现时间或“附近”状态。
- 连接按钮或整行点击。

操作：

- 下拉重新扫描。
- 点击设备连接。
- 连接中取消或返回。
- 失败后重试。
- 点击返回首页时不停止扫描，除非平台限制或产品连接模型主动停止扫描。

### 7.2 CAN 节点选择页

显示条件：

- 仅 Express 模块连接成功后显示。

字段：

- 节点 ID。
- 设备名称或硬件名称。
- 固件版本。
- UUID。
- 最近在线状态。

操作：

- 点击节点进入设备详情和实时页。
- 切换节点。
- 重新扫描节点。
- 返回设备列表。
- 断开 Express 模块。

### 7.3 设备信息页

归属：

- 设备信息优先作为设备 Tab 内的详情区或二级页。
- 实时数据页可以展示设备信息摘要，但不替代设备 Tab 的管理入口。

字段：

- 设备显示名称。
- BLE 名称。
- 节点 ID，如适用。
- 固件版本号。
- 软件版本号。
- UUID。
- 连接状态。
- 最近连接时间。

首版不提供：

- 固件升级按钮。
- 原始参数编辑。
- 终端。
- Bootloader。
- 工程调试入口。

### 7.4 实时数据页

入口：

- UART 直连设备连接成功并读取基础信息后自动进入。
- Express 模块选择 CAN 节点并读取基础信息后自动进入。
- 首页“查看实时数据”按钮进入。

字段：

- 速度。
- 电量百分比。
- 里程。
- 最高时速。
- 故障提示。

交互：

- 大号速度仪表居中。
- 故障提示优先级高于普通状态文案。

- 页面返回后降低轮询频率。
- 返回首页或设备 Tab 时保持连接。

## 9. 数据字段定义

| 字段 | 用户显示 | 来源建议 | 备注 |
| --- | --- | --- | --- |
| connectionState | 连接状态 | 产品连接模型 | 未连接/扫描中/连接中/已连接/错误 |
| navigationState | 导航状态 | 产品连接模型 | 驱动首页、设备、实时页之间跳转 |
| bleName | BLE 名称 | BLE 扫描结果/连接信息 | UART 直连设备内部兼容识别名固定为 `VESC BLE UART`；用户界面不直接展示该工程名 |
| firmwareVersion | 固件版本号 | FW_RX_PARAMS | 已有基础能力 |
| softwareVersion | 软件版本号 | `COMM_FW_VERSION` 响应中的 `FW_RX_PARAMS.major/minor`，可选拼接 `fwName` | 代码中由 `Commands::getFwVersion()` 请求并在 `Commands::processPacket()` 解析；CAN 节点可用 `Utility::getFwVersionBlockingCan()` 读取 |
| uuid | UUID | VescInterface connected UUID | iOS 不依赖 BLE MAC |
| speed | 实时速度 | SETUP_VALUES.speed | UI 转换 km/h 或 mph |
| batteryPercent | 电池电量百分比 | SETUP_VALUES.battery_level | 0-100 |
| odometer | 里程 | SETUP_VALUES.odometer 或统计字段 | 单位转换需统一 |
| maxSpeed | 最高时速 | 产品模型根据本次连接会话内的 `speed` 最大值计算 | 断开或重新连接后重置，不读取设备历史最高速度 |
| faultText | 故障提示 | SETUP_VALUES.fault_str 转用户文案 | 不直接显示晦涩代码 |
| telemetryValid | 数据有效 | 产品模型时间戳 | 超过 2 秒无数据为 false |

软件版本号协议来源：

- `commands.cpp` 中 `Commands::getFwVersion()` 发送 `COMM_FW_VERSION`。
- `Commands::processPacket()` 在收到 `COMM_FW_VERSION` 后解析 `FW_RX_PARAMS.major`、`minor`、`hw`、`uuid`、`hwType`、`fwName` 等字段。
- `datatypes.h` 中 `FW_RX_PARAMS` 已将 `major`、`minor`、`fwName`、`hw`、`uuid` 声明为可给 QML 使用的属性。
- `product/productdevicemodel.cpp` 当前已有 `firmwareVersion()`，使用 `FW_RX_PARAMS.major/minor` 组合为版本号。
- PRD 中“软件版本号”首版可与固件版本展示同源；如未来 BM 硬件另有独立软件版本字段，再新增产品字段，不阻塞 MVP。

## 10. 非功能需求

### 10.1 平台

- iOS：优先验证，目标满足 App Store 审核。
- Android：同一 QML 产品层适配，目标满足主流安卓应用市场审核。

### 10.2 性能

- 扫描页滚动流畅。
- 实时页高频轮询不应造成明显 UI 卡顿。
- 后台/非实时页面降低轮询频率，减少 BLE 压力和电量消耗。

### 10.3 稳定性

- 连接失败可恢复。
- 断连后 UI 状态一致。
- 超时、权限拒绝、蓝牙关闭、设备离线均有明确状态。

### 10.4 合规

- 开源许可入口必须可访问。
- 隐私政策必须说明蓝牙、定位权限使用原因。
- Android BLE 扫描涉及定位权限时，需要给出用户可理解说明。
- iOS `Info.plist` 蓝牙权限文案需使用 BM 品牌语言。
- App Store / 安卓市场截图和描述不得误用 VESC 商标。

首版隐私最低要求：

- 不要求账号、手机号、邮箱、实名信息或云端设备绑定。
- 不采集、不上传、不出售用户个人数据。
- 蓝牙权限仅用于发现和连接附近 BM 设备。
- 若 Android 系统或应用市场因 BLE 扫描要求定位相关权限，隐私政策和权限弹窗必须说明：该权限仅用于扫描附近蓝牙设备，不用于获取或记录用户地理位置。
- 崩溃日志、诊断日志如后续接入，需要单独说明采集内容、用途、保存周期和关闭方式；首版不应默认上传包含 UUID、位置、联系人或账号信息的日志。

## 13. 验收清单

### 13.1 设备发现与连接

- 打开 App 后自动开始 BLE 扫描。
- 默认停留首页摘要，完整设备列表只在设备 Tab 展示。
- 下拉可以重新扫描。
- 能显示附近设备列表。
- 点击设备后进入连接中。
- 15 秒连接超时后提示并可重试。
- BLE 错误显示为用户可理解文案。
- 连接成功后全局状态更新。
- 连接失败、超时、取消后回到设备 Tab，并保留重试入口。

### 13.2 导航与页面职责

- 底部 Tab 仅包含首页、设备、我的。
- 首页不展示完整 BLE 扫描列表。
- 设备 Tab 是扫描、连接、节点选择、设备详情的唯一入口。
- 我的 Tab 不出现登录、注册、云同步、社区、排行榜、VIP 调参。
- 切换 Tab 不导致 BLE 断开。
- 断开成功后回到首页未连接态，并清空或置灰遥测数据。

### 13.3 Express 模块

- 识别 Express 蓝牙设备。
- 显示 CAN 节点设备列表。
- 点击节点后显示固件版本号、软件版本号、UUID。
- 进入对应节点实时数据页。
- 下次连接同一 Express 模块时按保存的 CAN 节点 ID 默认选择上次节点。
- 保存的 CAN 节点 ID 不存在或不可用时回到节点列表。
- 切换节点时不展示上一个节点的旧遥测数据。

### 13.4 UART 直连设备

- 识别 UART 直连设备。
- 连接后直接显示固件版本号、软件版本号、UUID。
- 自动进入实时数据页。

### 13.5 遥测

- 实时速度更新正常。
- 电池电量百分比更新正常。
- 里程显示正常。
- 最高时速显示正常。
- 故障提示显示正常。
- 实时页使用高频轮询。
- 非实时页使用低频轮询。
- 超过 2 秒未收到数据后标记遥测失效。
- 最高时速按本次连接会话实时速度最大值计算，断开或重新连接后重置。

### 13.6 退出与断连

- 退出 App 前主动断开 BLE。
- App 进入后台时立即主动断开 BLE。
- 断开后 UI 不保留假连接状态。
- 再次进入可重新扫描连接。

### 13.7 合规与品牌

- 用户可见 App 名称为 BM。
- 首版 UI 不出现 VESC 商标，除必要开源/兼容说明。
- 设置页包含隐私政策、用户协议、支持、开源许可入口。
- iOS 与 Android 权限说明符合商店审核。
- 隐私政策说明蓝牙权限用途；Android 如要求定位相关权限，需说明仅用于 BLE 扫描，不用于记录地理位置。

## 14. 主要风险

- 首页、设备 Tab、实时数据页若各自维护状态，容易造成重复连接、错误跳转或断连后 UI 假在线。
- 如果继续使用“发现”命名，用户和实现都可能把社区、活动、排行榜等第二阶段功能带回首版。
- 现有 `ConnectScreen.qml` 仍包含较多历史工程逻辑，直接复用可能把工程概念带回首版 UI。
- Express 模块与 CAN 节点选择可能需要更清晰的产品层模型，否则 QML 容易直接调用底层 Commands。
- iOS 与 Android BLE 权限模型不同，自动扫描体验可能不完全一致。
- 如果高频轮询过快，可能影响 BLE 稳定性和手机耗电，需要真机调参。
- 进入后台立即断开 BLE 会提升连接占用安全性，但用户切回前台时需要重新连接，需用清晰状态文案降低困惑。
- 原型 HTML 含有社区、登录、升级、VIP、VESC 文案，不能原样进入产品。
